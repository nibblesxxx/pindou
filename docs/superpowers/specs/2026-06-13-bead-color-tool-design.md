# 拼豆色号管理工具 — 设计文档

## 概述

一个部署在 GitHub Pages 的纯前端 Web 工具，通过 GitHub API 读写仓库 CSV 文件，实现手机和电脑数据同步。帮助拼豆爱好者管理 MARD 色号库存，支持上传拼豆截图自动匹配色号并扣减库存。

## 技术栈

| 项 | 选择 |
|---|---|
| 前端框架 | 无框架，纯 HTML + CSS + JS，单文件 `index.html` |
| MARD 色号数据 | 硬编码内置（291 色，来源 `mard.csv`） |
| 库存持久化 | GitHub API 读写仓库中的 `warehouse.csv` |
| 部署 | GitHub Pages（免费） |
| 认证 | GitHub Personal Access Token（存在 localStorage） |
| 图片取色 | Canvas API，逐像素提取颜色 |
| 颜色匹配 | RGB 欧几里得距离最小化，匹配最近 MARD 色号 |
| 本地降级 | 无 Token 时退化为 localStorage 模式（单设备使用） |

## 核心原则

- **云端 CSV 是仓库本体**：库存数据存储在 GitHub 仓库的 `warehouse.csv` 中
- **手动保存到云端**：库存变动后，用户点击保存，通过 GitHub API 写回 `warehouse.csv`
- **多设备同步**：手机和电脑打开同一个 GitHub Pages 网址，共享同一份 CSV 数据
- **简洁优先**：两个 Tab 界面，白色背景，每个色号用真实颜色标注

## 项目文件结构

```
<GitHub 仓库>/
├── index.html          # 应用文件（GitHub Pages 自动托管）
├── warehouse.csv       # 用户库存数据（GitHub API 读写）
├── mard.csv            # MARD 色号对照表（已存在，参考用）
└── README.md           # 使用说明
```

## 界面设计

### 界面一：色号仓库

**功能：**
- 色块网格展示全部 291 个 MARD 色号，每个色块显示真实 RGB 颜色、色号名称、库存数量
- 搜索框过滤色号
- 点击色块弹出对话框修改库存数量
- 修改后自动标为"未保存"，用户需点击「💾 保存到云端」写回 GitHub
- 「🔄 从云端加载」按钮：从 GitHub 拉取最新 `warehouse.csv`
- 库存 0 的色号灰色显示，低库存高亮提醒

**CSV 格式：**
```
色号,库存
A1,500
A10,200
...
```

**空库存初始化：** 首次使用（无 `warehouse.csv` 或库存全空）时自动生成全 0 模板

### 界面二：图转色号

**功能：**
- 拖拽 / 点击上传拼豆截图（PNG / JPG / WebP）
- Canvas 提取所有不同颜色（去重）
- 每个颜色 RGB 值与 291 个 MARD 色号计算欧几里得距离，匹配最近色号
- 统计每种匹配色号的像素数量
- 展示匹配结果：色块 + 色号 + 数量 + 库存状态（不足标黄）
- 库存不足允许扣除（变负数，标红提醒）
- ✅ 确认扣除：从库存中减去对应数量
- ↩ 撤销上次扣除：恢复库存到扣除前状态（记录快照）
- 扣除后提示保存到云端
- 支持重新识别（换图或用同一张图重试）

**颜色匹配算法：**
```
对于图片中每种不同颜色 (r, g, b):
  最近色号 = min(MARD色号, distance(rgb, mard_rgb))
  distance = sqrt((r-R)^2 + (g-G)^2 + (b-B)^2)
统计每种匹配色号的像素数量 → 展示结果
```

### 导航

- 顶部两个 Tab 切换：「📦 色号仓库」「🎨 图转色号」
- 底部状态栏显示连接状态（云端/本地模式）

## 数据流

```
GitHub 仓库 warehouse.csv
    ↕ GitHub API (手动触发 加载/保存)
网页 localStorage (工作副本 + Token + 撤销快照)
    ↕ 用户操作
界面展示
```

## 首次配置流程

1. 用户在 GitHub 创建 Personal Access Token（仅需 `repo` 权限）
2. 打开 GitHub Pages 网址
3. 页面自动检测 Token 缺失 → 弹出**配置引导框**
4. 引导框两个输入项：
   - GitHub Token（密码框，不可见）
   - 仓库路径（如 `username/repo`）
5. 可选：提供"如何创建 Token"的步骤说明，附带 GitHub 直达链接
6. 保存后 Token 写入 localStorage，自动跳转主界面
7. 点击「🔄 从云端加载」拉取 `warehouse.csv`
8. 每次打开页面自动检测 Token，有效则跳过引导框

**引导框触发条件：**
- 首次访问（localStorage 无 Token）
- Token 失效或手动清除缓存后 → 重新弹出
- 已保存 Token 则自动跳过

**未配置 Token 时：** 降级为纯本地模式（localStorage 存储），无多设备同步。用户可随时在设置中重新配置。

## 撤销机制

1. 扣除前：将当前库存状态保存为快照
2. 扣除后：用户可点击「撤销上次扣除」
3. 撤销：恢复快照（不影响云端，需手动保存才同步到 GitHub）
4. 只保存最近一次扣除的快照（新的扣除覆盖旧快照）

## 边界与错误处理

- 无 Token / Token 过期：降级为本地模式，底部状态栏提示
- GitHub API 限流：个人使用不会触发，如触发则提示稍后重试
- 图片过大：限制 Canvas 处理尺寸为最大 2000×2000px，超出自动缩放
- 颜色数量过多（超过 500 种不同颜色）：提示用户确认是否为有效拼豆图
- 库存负数：标红显示，提醒补货
- 网络错误：保存失败时保留本地修改，提示重试

## 不做的事情

- 不建后端服务器
- 不用任何前端框架
- 不处理实际的豆子（只是数字管理）
- 不支持 WeChat 小程序（纯 Web）
- 不需要用户登录/注册（仅需 GitHub PAT）
- 不做自动保存（手动保存避免误操作覆盖云端数据）
